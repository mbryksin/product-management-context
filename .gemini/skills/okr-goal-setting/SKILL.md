# OKR and goal setting

## When to use

- Setting or reviewing OKRs at company, product, or team level.
- Need to separate outcomes from initiatives and align metrics with analytics/KPI guidance.

## Steps

1. Read [`01_strategy/okrs/_overview.md`](../../../.productmap/01_strategy/okrs/_overview.md), [`09_templates/okr-template.md`](../../../.productmap/09_templates/okr-template.md), and [`08_frameworks/okrs.md`](../../../.productmap/08_frameworks/okrs.md).
2. Define period with absolute dates; for each KR include metric definition, baseline, target, owner, and data source.
3. Add alignment table to higher-level goals when hierarchy is known; note confidence and gaming risks.

## Delegation

Use [subagent-prompt.md](subagent-prompt.md) for a sub-agent that owns full OKR drafting and critique.
